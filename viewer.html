⚠ Critical Issue
Skills are injected at role: "user" level instead of system level.

This results in 53% performance — same as no instructions. To achieve 100%, skills must be moved to the system prompt.

⚡ Where Skills Get Injected
When Claude invokes a skill tool, the full skill content is inserted into the messages array as a user message. This is low-authority and competes with user context.
messages array — After Skill tool invocation
{ "role": "user", "content": [{ "type": "text", "text": "<system-reminder> The 'pdf' skill is now active. Base directory: /sessions/.../skills/pdf </system-reminder> # PDF Processing Skill ## Overview Comprehensive PDF document manipulation: - Reading and extracting text - Creating new PDF documents - Merging multiple PDFs ... ## Instructions ### Library Selection - Use pypdf2 for basic operations - Use pdfrw for form filling - Use reportlab for creating PDFs [~18KB of detailed instructions]" }] }
The Problem
// As a USER message, this competes with: // - User's actual task description // - Context about what user wants // - Other important instructions // If the model has 200k context and skill is 18k, // the skill dilutes other context by ~9% // But worse: system-level instructions have // 3-5x more influence than user-level content
system array — Claude Code System Prompt
{ "system": [ { "type": "text", "text": "[billing header]", "cache_control": { "type": "ephemeral" } }, { "type": "text", "text": "You are Claude Code, a specialized agent for the Claude Code IDE. You have access to tools for file manipulation, execution... [~140KB of foundational instructions]", "cache_control": { "type": "ephemeral" } } // NOTE: Skills are NOT here! ] }
SYSTEM PROMPT AUTHORITY
The system prompt is the most authoritative source of instructions. Every token here shapes the model's behavior for the entire conversation. Moving skill content here would result in 100% compliance instead of current 53%.
tools array — Available Tools & Skills
{ "tools": [ { "name": "Read", ... }, { "name": "Write", ... }, { "name": "Bash", ... }, { "name": "Skill", "description": "Execute specialized skills Available skills: - pdf: PDF processing - xlsx: Excel/spreadsheet tools - docx: Word documents - pptx: PowerPoint presentations ..." // Only metadata, not full instructions }, // ~80 tools total ] }
TOOL ARRAY AUTHORITY
The Skill tool has only a brief description in the tools array. The full skill instructions are NOT here — they're injected later as user messages when the tool is invoked. This delays critical context until after the tool decision is made.
Request Flow Diagram
1
User requests task

2
Model sees Skill tool in tools array
Only sees brief description: "Execute specialized skills"
3
Model invokes skill
Calls: {"name": "Skill", "input": {"skill": "pdf"}}
4
⚠ Skill content injected as USER message
Full 18KB of skill instructions inserted into messages array as "role: user"
5
Model processes with low authority
Result: 53% compliance (same as no instructions)
Solution
Move skill instructions to the system prompt at initialization, not as user messages on invocation. This gives skills the authority level they need for 100% compliance.
